### Postman: Тестирование и отладка API

Postman — это мощный инструмент для работы с API, который позволяет отправлять запросы к серверу, просматривать ответы и автоматизировать тестирование.

#### Основные возможности:
1. **Создание запросов:** Поддержка всех HTTP-методов (GET, POST, PUT, PATCH, DELETE).
2. **Работа с телом запроса (Body):** Удобный интерфейс для отправки JSON-данных.
3. **Коллекции:** Возможность группировать запросы по темам или проектам.
4. **Переменные:** Использование окружений (Environments) для быстрого переключения между локальным сервером и продакшеном.

#### Инструкция по обновлению данных (PUT):
В нашем проекте обновление автомобиля реализовано через метод **PUT**, что подразумевает полную замену объекта.

**Как отправить запрос:**
1. Выберите метод `PUT`.
2. URL: `http://localhost:8080/api/cars/1`.
3. Вкладка **Body** -> **raw** -> в выпадающем списке выберите **JSON**.
4. Введите полный объект (если пропустить поле, оно может затереться в базе данных `null` или `0`).

**Пример JSON:**
```json
{
    "brand": "BMW",
    "model": "X5 M-Sport",
    "productionYear": 2021,
    "mileage": 45000,
    "price": 55000,
    "status": "AVAILABLE",
    "color": "Midnight Blue",
    "horsepower": 340,
    "fuelType": "DIESEL",
    "transmission": "AUTOMATIC"
}
```

#### PUT vs PATCH:
- **PUT:** Полное обновление ресурса. Нужно отправлять все поля.
- **PATCH:** Частичное обновление. Отправляются только те поля, которые нужно изменить.
*Примечание: В текущей реализации проекта используется PUT.*

#### Особенности и решение проблем:
1. **H2 In-Memory Database:** Помните, что при перезагрузке приложения в IntelliJ IDEA все изменения в базе данных стираются, так как база живет в оперативной памяти.
2. **Кэширование:** Если после PUT-запроса в общем списке `GET /api/cars` вы видите старые данные, а по ID `GET /api/cars/1` — новые, попробуйте:
   - Нажать кнопку **Send** еще раз.
   - Использовать **Send and Download**, чтобы исключить кэш Postman.
   - Проверить результат в браузере.
3. **JSON Format:** Всегда проверяйте наличие запятых и правильность кавычек в JSON-теле запроса.
