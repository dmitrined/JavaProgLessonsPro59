### Apache Maven: Сборка и управление зависимостями

Maven — это инструмент для автоматизации сборки Java-проектов, управления зависимостями и отчетности.

#### 1. Жизненный цикл (Lifecycle) и фазы
Maven основан на концепции жизненного цикла. Каждая фаза выполняет определенную задачу. Выполнение фазы также запускает все предшествующие ей фазы.

* **clean** — удаляет директорию `target`, очищая результат предыдущих сборок.
* **validate** — проверяет, корректен ли проект и доступна ли вся необходимая информация.
* **compile** — компилирует исходный код проекта (`src/main/java` -> `target/classes`).
* **test** — запускает unit-тесты (обычно с помощью JUnit).
* **package** — упаковывает скомпилированный код в формат, пригодный для распространения (например, JAR или WAR).
* **verify** — выполняет проверки для подтверждения качества пакета и соответствия критериям.
* **install** — устанавливает пакет в локальный репозиторий (~/.m2/repository), чтобы его можно было использовать как зависимость в других локальных проектах.
* **site** — генерирует документацию и отчеты по проекту.
* **deploy** — копирует финальный пакет на удаленный репозиторий для совместного использования с другими разработчиками.

#### 2. Полезные команды
* `mvn clean package` — классическая команда для сборки проекта «с нуля».
* `mvn dependency:tree` — выводит дерево зависимостей, помогая найти конфликты версий библиотек.
* `mvn test` — запуск только тестов.
* `./mvnw` (Maven Wrapper) — позволяет запускать Maven без его предварительной установки в систему.

#### 3. Файл pom.xml
`Project Object Model` — основной файл настроек Maven, где описываются:
* Координаты проекта (`groupId`, `artifactId`, `version`).
* Зависимости (`dependencies`).
* Плагины (`build plugins`).
* Свойства (`properties`).
