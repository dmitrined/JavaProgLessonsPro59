Инструкция: Как сделать Push в GitHub

1. Проверить статус изменений
Команда: git status
Зачем: Показывает, какие файлы были изменены, добавлены или удалены.

2. Добавить изменения в индекс (Stage)
Команда: git add .
Зачем: Подготавливает все измененные файлы для коммита. Если нужно добавить конкретный файл: git add <путь_к_файлу>.

3. Создать коммит (Зафиксировать изменения)
Команда: git commit -m "Ваше описание изменений"
Зачем: Сохраняет изменения в локальном репозитории с комментарием.

4. Отправить изменения в удаленный репозиторий (GitHub)
Команда: git push
Зачем: Переносит ваши локальные коммиты на сервер GitHub.

Дополнительные команды:
- git pull — затянуть свежие изменения из GitHub в локальный проект (делать перед началом работы).
- git remote -v — проверить, к какому репозиторию привязан проект.
