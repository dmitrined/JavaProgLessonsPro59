### Объектно-Ориентированное Программирование (ООП)

ООП — это методология программирования, основанная на представлении программы в виде совокупности объектов, каждый из которых является экземпляром определенного класса.

#### Основные понятия:
*   **Класс** — шаблон или чертеж, описывающий состояние (поля) и поведение (методы) объектов.
*   **Объект** — конкретный экземпляр класса, созданный в памяти.
*   **Конструктор** — специальный метод, вызываемый при создании объекта (`new`). Используется для инициализации полей.

#### 4 Столпа ООП:

1.  **Инкапсуляция**
    *   Сокрытие внутренней реализации объекта и защита данных от прямого доступа извне.
    *   Достигается с помощью модификаторов доступа (`private`) и публичных методов доступа (геттеры и сеттеры).
    *   Пример: Поле `private int age` доступно только через метод `setAge(int age)`, где можно добавить проверку (валидацию).

2.  **Наследование**
    *   Механизм, позволяющий создавать новый класс (потомок) на основе существующего (родитель).
    *   Ключевое слово: `extends`.
    *   Потомок наследует все поля и методы родителя (кроме `private`), может добавлять свои или переопределять родительские.
    *   В Java поддерживается только **одиночное** наследование классов.
    *   Все классы неявно наследуются от класса `Object`.

3.  **Полиморфизм**
    *   Способность объектов с одинаковой спецификацией иметь различную реализацию.
    *   **Статический (Compile-time):** Перегрузка методов (Overloading) — методы с одним именем, но разными параметрами.
    *   **Динамический (Runtime):** Переопределение методов (Overriding) — изменение реализации метода родительского класса в дочернем (аннотация `@Override`).

4.  **Абстракция**
    *   Выделение значимых характеристик объекта и игнорирование деталей реализации.
    *   **Абстрактный класс (`abstract class`):** Может содержать абстрактные методы (без тела) и обычные методы. Нельзя создать экземпляр абстрактного класса.
    *   **Интерфейс (`interface`):** Контракт, который класс обязуется выполнить (`implements`). Содержит только константы и абстрактные методы (до Java 8). С Java 8 может содержать `default` и `static` методы. Поддерживается множественная реализация интерфейсов.

#### Модификаторы доступа:
*   `private`: доступ только внутри класса.
*   `default` (package-private): доступ внутри пакета (если модификатор не указан).
*   `protected`: доступ внутри пакета и в классах-наследниках (даже в других пакетах).
*   `public`: доступ из любого места программы.

#### Ключевое слово `static`:
Принадлежит классу, а не экземпляру.
*   `static` поле: общее для всех объектов класса.
*   `static` метод: можно вызвать без создания объекта (`ClassName.method()`). Не имеет доступа к `this`.
