### Коллекции в Java (Collections Framework)

Java Collections Framework — это набор интерфейсов и классов для хранения и обработки групп объектов. Находится в пакете `java.util`.

#### Иерархия интерфейсов:
`Iterable` -> `Collection` -> (`List`, `Set`, `Queue`)
`Map` (не наследуется от Collection, но является частью фреймворка)

#### 1. List (Список)
Упорядоченная коллекция, допускает дубликаты. Доступ к элементам по индексу.
*   **ArrayList:**
    *   Реализован на основе динамического массива.
    *   Быстрый доступ по индексу (`get`) — O(1).
    *   Медленная вставка/удаление в середине (сдвиг элементов) — O(n).
    *   Хорош для чтения данных.
*   **LinkedList:**
    *   Двусвязный список.
    *   Быстрая вставка/удаление элементов — O(1) (если есть итератор).
    *   Медленный доступ по индексу (перебор) — O(n).
    *   Реализует также интерфейсы `Queue` и `Deque`.

#### 2. Set (Множество)
Коллекция уникальных элементов (без дубликатов).
*   **HashSet:**
    *   Основан на хеш-таблице (`HashMap`).
    *   Порядок элементов не гарантируется.
    *   Самый быстрый (O(1) для основных операций).
    *   Требует корректного переопределения `equals()` и `hashCode()`.
*   **LinkedHashSet:**
    *   Сохраняет порядок вставки элементов.
    *   Чуть медленнее `HashSet`.
*   **TreeSet:**
    *   Хранит элементы в отсортированном виде (Natural ordering или Comparator).
    *   Основан на красно-черном дереве (`TreeMap`).
    *   Медленнее (O(log n)).

#### 3. Map (Карта / Словарь)
Хранит пары "ключ-значение" (Key-Value). Ключи уникальны.
*   **HashMap:**
    *   Порядок не гарантируется.
    *   Разрешает `null` в качестве ключа и значений.
*   **LinkedHashMap:**
    *   Сохраняет порядок вставки (или доступа).
*   **TreeMap:**
    *   Ключи отсортированы.

#### 4. Queue (Очередь) и Deque (Дек)
*   **PriorityQueue:** Очередь с приоритетом (элементы сортируются).
*   **ArrayDeque:** Двусторонняя очередь (быстрее `LinkedList` и `Stack`).

#### Класс Collections
Утилитный класс с статическими методами:
*   `Collections.sort(list)` — сортировка.
*   `Collections.shuffle(list)` — перемешивание.
*   `Collections.reverse(list)` — разворот.
*   `Collections.binarySearch(list, key)` — бинарный поиск (в отсортированном списке).

#### Итератор (Iterator)
Объект для перебора коллекции. Позволяет безопасно удалять элементы во время итерации.
```java
Iterator<String> iter = list.iterator();
while (iter.hasNext()) {
    String s = iter.next();
    if (s.equals("delete me")) {
        iter.remove();
    }
}
```
