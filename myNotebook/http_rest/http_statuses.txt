4) HTTP-статусы: коды ответов сервера

Статус-коды сообщают клиенту о результате выполнения его запроса. Они делятся на группы по первой цифре.

2xx — Успех (Success)
- **200 OK** — запрос успешно обработан. Обычно возвращается при GET и PUT.
- **201 Created** — ресурс успешно создан. Стандартный ответ на POST.
- **204 No Content** — запрос выполнен успешно, но в ответе нет тела. Часто используется при DELETE или обновлении, если клиент не ждет данных в ответ.

4xx — Ошибка клиента (Client Error)
- **400 Bad Request** — неверный формат запроса (ошибка в JSON, нарушение правил валидации @Valid, неверный тип Enum).
- **401 Unauthorized** — клиент не аутентифицирован (нужно войти в систему).
- **403 Forbidden** — доступ запрещен (недостаточно прав).
- **404 Not Found** — запрашиваемый ресурс не найден (например, машины с таким ID нет в базе).
- **405 Method Not Allowed** — ресурс не поддерживает данный метод (например, попытка сделать POST на URL, где разрешен только GET).
- **409 Conflict** — конфликт состояния ресурса (например, попытка добавить запись, которая уже существует, или изменить данные, которые в данный момент заблокированы).

5xx — Ошибка сервера (Server Error)
- **500 Internal Server Error** — общая ошибка на стороне сервера (необработанное исключение, падение БД).
- **503 Service Unavailable** — сервер временно не может обрабатывать запросы (перегрузка, техобслуживание).
