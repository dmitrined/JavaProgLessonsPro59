### Swagger UI: Интерактивная документация и тестирование API

Swagger UI — это инструмент, который автоматически генерирует визуальную страницу документации для вашего REST API на основе кода. Он превращает техническую спецификацию (OpenAPI) в удобный интерфейс.

**Ссылка:** http://localhost:8080/swagger-ui/index.html (доступна при запущенном приложении)

#### Основные функции:

1. **Документация «в реальном времени»:**
   - Показывает все доступные контроллеры и их эндпоинты (GET, POST, PUT, DELETE).
   - Автоматически обновляется при добавлении новых методов в Java-код.

2. **Интерактивное тестирование (Try it out):**
   - Позволяет отправлять запросы прямо из браузера без использования Postman.
   - Автоматически формирует шаблон JSON-тела для POST и PUT запросов.
   - Показывает реальные ответы сервера (статус-коды, заголовки, тело ответа).

3. **Описание моделей данных (Schemas):**
   - Визуализирует структуру ваших Entity (например, класс `Car`).
   - Указывает типы полей (String, Integer, Enum) и ограничения (например, валидацию).

4. **Песочница (Sandbox):**
   - Идеально подходит для фронтенд-разработчиков, чтобы понять, как взаимодействовать с бэкендом, не читая исходный код.

#### Почему это важно:
Swagger UI — это «лицо» вашего API. Он делает разработку прозрачной, упрощает отладку и служит всегда актуальной инструкцией для всех, кто будет пользоваться вашим сервисом.
