1) Контроллер в Spring Boot: зачем он нужен
Контроллер — верхний слой приложения (Web слой), который служит «точкой входа» в приложение для внешних клиентов (браузеров, мобильных приложений, других микросервисов).

Основные задачи контроллера:
- Принимает HTTP-запрос (URL + метод + заголовки + тело).
- Десериализует входные данные из JSON в Java-объекты (с помощью Jackson).
- Валидирует входные данные (используя Bean Validation, например @Valid).
- Вызывает методы сервисного слоя (Service) для выполнения бизнес-логики.
- Формирует и возвращает HTTP-ответ (статус-код + тело в формате JSON + заголовки).

❗ Контроллер НЕ должен содержать сложную бизнес-логику или напрямую работать с базой данных.
Его роль — оркестрация потока данных на границе приложения. Все бизнес-правила должны быть вынесены в сервисы.
